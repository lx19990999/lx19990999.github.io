使用docker在kylin desktop v10sp1 arm64环境中编译nginx

在linux.do论坛上看了个帖子，于是现学现卖用amd64的docker运行arm64镜像，整理如下：

编辑 Docker 配置文件 /etc/docker/daemon.json,添加以下内容以启用多平台支持：
{
“experimental”: true,
“features”: { “buildkit”: true }
}
重启 Docker 服务：sudo systemctl restart docker
Docker Desktop 内置了 QEMU，但需手动注册多架构支持：
docker run –rm –privileged multiarch/qemu-user-static –reset -p yes
docker镜像仅供参考，请使用可信源：
本地目录/home/flutter/docker/kylin/data/映射到docker的/data目录，并调整cpu、内存限制，执行命令前连接互联网：
docker run -it –name kylin -p 80:80 –cpus=8 –memory=32g -v /home/flutter/docker/kylin/data/:/data/ –platform linux/arm64 cnxc/kylin:desktop-v10sp1 /bin/bash
docker run -it –name kylin -p 80:80 –cpus=8 –memory=32g -v /c/dockervolume/kylin/data/:/data/ –platform linux/arm64 cnxc/kylin:desktop-v10sp1 /bin/bash
下载nginx-1.29.1.tar.gz、nginx-module-vts-0.2.4.zip到data目录
cd /data
tar -xvf nginx-1.29.1.tar.gz
unzip nginx-module-vts-0.2.4.zip
检查更新并安装编译环境：
apt update
apt install build-essential automake autoconf libtool libpcre3 libpcre3-dev zlib1g-dev libssl-dev -y
编译nginx安装到/data/nginx目录：
cd /data/nginx-1.29.1
./configure –prefix=/data/nginx –with-http_ssl_module –with-http_v2_module –with-http_stub_status_module –with-stream –add-module=/data/nginx-module-vts-0.2.4
make && make install
/data/nginx/sbin/nginx -V
打包nginx到生产环境测试：
tar -cvf nginx20250819.tar –exclude=’*.log’ /data/nginx
